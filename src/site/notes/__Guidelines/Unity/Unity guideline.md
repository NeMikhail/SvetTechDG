## Требования

Для работы с проектом используется Unity `6000.3.8f1`.

Перед открытием проекта необходимо установить:

1. Unity Hub.
2. Unity Editor версии `6000.3.8f1`.
3. Модули Unity, необходимые для целевой платформы проекта.
4. Git-клиент, если проект еще не загружен локально.

## Установка Unity Hub

<details> <summary>🎬 Видео-инструкция к этому этапу</summary> <br><iframe src="https://drive.google.com/file/d/13-Un1gVnPS0FFI6VBWTi9Elgi5eHg8H7/preview" width="640" height="480"></iframe></details>

Unity Hub используется для установки редактора Unity, управления версиями Unity и открытия локальных проектов.

1. Установите Unity Hub с официального сайта Unity.
2. Войдите в Unity ID
3. При необходимости настройте путь установки редакторов Unity:
   - откройте настройки Unity Hub;
   - перейдите в раздел установки редакторов;
   - укажите директорию, в которой будут храниться версии Unity.

## Установка версии Unity

<details> <summary>🎬 Видео-инструкция к этому этапу</summary> <br> <iframe src="https://drive.google.com/file/d/1YxujkCRXHr8DdufUfjmtDmLV_iOSX_05/preview" width="640" height="480"></iframe></details>

Проект должен открываться в Unity `6000.3.8f1`.

1. Откройте Unity Hub.
2. Перейдите в раздел установки редакторов.
3. Установите версию `6000.3.8f1` из [архива](unityhub://6000.3.8f1/1c7db571dde0). 

Если Unity Hub предлагает открыть проект в другой версии редактора, не соглашайтесь на автоматическое обновление версии проекта.

## Загрузка проекта из Git

<details> <summary>🎬 Видео-инструкция к этому этапу</summary> <br> <iframe src="https://drive.google.com/file/d/1fR4WGOlozuC8w0ClCKHOQLb6iEn6R3Kk/preview" width="640" height="480"></iframe></details>

Перед открытием проекта в Unity необходимо получить актуальную копию репозитория.

1. Склонируйте репозиторий через Git-клиент или командную строку.
2. Убедитесь, что выбрана нужная ветка.
3. Дождитесь полной загрузки файлов репозитория.
## Открытие проекта

1. Откройте Unity Hub.
2. Нажмите `Add` > `Add project from disk`.
3. Выберите корневую директорию проекта, загруженного из Git.
4. Убедитесь, что для проекта выбрана Unity `6000.3.8f1`.
5. Откройте проект и дождитесь окончания импорта ассетов.

Первый импорт может занять продолжительное время. До завершения импорта не рекомендуется закрывать Unity.

## Проверка после открытия

После первого открытия проекта необходимо выполнить базовую проверку.

1. Убедитесь, что Unity открылась без критических ошибок в Console.
2. Проверьте, что зависимости из `Packages` установились корректно.
3. Откройте основную рабочую сцену проекта, если она указана в документации команды.
4. Запустите проект в Play Mode и проверьте, что стартовая сцена запускается без ошибок.

Если после открытия появились ошибки импорта, отсутствующие пакеты или предупреждение о несовпадении версии Unity, необходимо сообщить команде и не фиксировать автоматические изменения проектных файлов без проверки.
