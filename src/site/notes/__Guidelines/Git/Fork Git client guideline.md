Официальный сайт Fork: https://git-fork.com/

## Установка

- Скачать Fork с официального сайта.
- Установить приложение с настройками по умолчанию.
- Запустить Fork.
- Если Fork попросит выбрать Git, использовать установленный системный Git.
- Если Git еще не установлен, сначала выполнить первый шаг [[__Guidelines/Git/Git guideline\|Git guideline]].

## Первичная настройка

- Открыть настройки Fork.
- Проверить имя пользователя и email для коммитов.
- Имя и email должны совпадать с настройками Git:

```bash
git config --global user.name
git config --global user.email
```

- Для клонирования проекта использовать SSH-ссылку из GitLab.

## Клонирование проекта

- Открыть страницу репозитория в GitLab.
- Нажать кнопку `Code`.
- Выбрать вариант `Clone with SSH`.
- Скопировать SSH-ссылку на репозиторий.
- В Fork выбрать `File` -> `Clone`.
- Вставить SSH-ссылку в поле `Repository URL`:

```text
ssh://git@gitlab-games.elementsoftware.ru:2222/svet_mmorpg/svet_mmo.git
```

- В поле `Parent Folder` выбрать папку, куда будет загружен проект.
- В поле `Name` оставить название проекта или указать удобное локальное имя папки.
- Нажать `Clone`.
- Дождаться завершения загрузки.
- Убедиться, что репозиторий открылся в Fork.
- Открыть папку проекта на компьютере и проверить, что файлы проекта скачались.
- Если Fork просит логин и пароль, значит была скопирована HTTPS-ссылка. Нужно вернуться в GitLab и скопировать `Clone with SSH`.

## Начало работы над задачей

- Перед началом задачи перейти на целевую ветку.
- Выполнить `Pull`, чтобы получить последние изменения.
- Создать новую ветку под задачу.
- Название ветки должно соответствовать [[__Conventions/1. Git convention\|1. Git convention]].
- Убедиться, что новая ветка выбрана как текущая.

## Работа с изменениями

- После правок открыть Fork и посмотреть вкладку с измененными файлами.
- Проверить, что в список изменений не попали временные файлы, локальные настройки IDE и лишние ассеты.
- Выбрать только файлы, которые относятся к задаче.
- Написать короткое английское сообщение коммита.
- Создать коммит.
- Если изменения логически разные, разбить их на несколько коммитов.

## Отправка изменений

- Перед отправкой убедиться, что выбрана правильная ветка.
- Нажать `Push`.
- Если Fork предлагает опубликовать новую ветку, подтвердить публикацию.
- При завершении задачи открыть GitLab и создать pull request.
- Описание pull request заполнять по правилам из [[__Guidelines/Git/Git guideline#7. Merge request\|Git guideline#7. Merge request]].

## Обновление ветки

- Перед продолжением работы нажать `Pull`.
- Если ветка отстала от целевой ветки, выполнить rebase на целевую ветку перед merge.
- Если при rebase появились конфликты, не решать конфликты в Unity-сценах, prefab или крупных asset-файлах без согласования.
- После решения конфликтов проверить проект.
- Отправить обновленную ветку через `Push`.
