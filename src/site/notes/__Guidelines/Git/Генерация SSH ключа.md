## Когда нужен SSH-ключ

- SSH-ключ нужен, чтобы Git мог подключаться к GitLab без ввода логина и пароля.
- Ключ создается один раз для конкретного компьютера.
- Публичная часть ключа добавляется в GitLab.
- Приватную часть ключа нельзя отправлять в чат, выкладывать в репозиторий или передавать другим людям.
## Генерация ключа

- Открыть терминал. Поиск -> cmd
- Выполнить команду, заменив email на свой рабочий:

```bash
ssh-keygen -t ed25519 -C "email@example.com"
```

- Если `ed25519` не поддерживается, использовать RSA:

```bash
ssh-keygen -t rsa -b 4096 -C "email@example.com"
```

- На вопрос о пути сохранения можно оставить путь по умолчанию.
- Passphrase оставить пустым.
- После генерации проверить, что появились файлы ключа:

```bash
ls ~/.ssh
```

## Как найти публичный ключ

- Публичный ключ обычно лежит в одном из файлов:

```text
~/.ssh/id_ed25519.pub
~/.ssh/id_rsa.pub
```

- Содержимое `.pub` файла нужно скопировать целиком.
- Файл без `.pub` является приватным ключом. Его копировать в GitLab нельзя.

## Проверка подключения

- После добавления ключа в GitLab выполнить:

```bash
ssh -T git@gitlab-games.elementsoftware.ru
```

- При первом подключении можно подтвердить добавление хоста в known hosts.

## Что делать при ошибке

- Проверить, что в GitLab добавлен именно публичный ключ с расширением `.pub`.
- Проверить, что ключ лежит в стандартной папке `~/.ssh`.
- Если ошибка остается, передать ответственному текст ошибки и вывод команды:

```bash
ssh -T git@gitlab-games.elementsoftware.ru
```
