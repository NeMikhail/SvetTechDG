## Зачем включать 2FA

- Двухфакторная аутентификация защищает аккаунт, даже если пароль стал известен другому человеку.
- При входе GitLab запрашивает пароль и второй фактор подтверждения.
- Вторым фактором может быть код из приложения-аутентификатора, WebAuthn-устройство или код из email.
- Для рабочего аккаунта рекомендуется использовать приложение-аутентификатор.

Официальная документация GitLab: https://docs.gitlab.com/user/profile/account/two_factor_authentication/

## Что подготовить заранее

- Убедиться, что есть доступ к email, указанному в GitLab.
- Установить приложение-аутентификатор на телефон.
- Подготовить место для хранения recovery codes.

Подходящие приложения:

- Authy.
- Microsoft Authenticator.
- Google Authenticator.
- Aegis Authenticator.

## Включение 2FA через приложение-аутентификатор

- Открыть GitLab.
- В правом верхнем углу нажать на аватар профиля.
- Выбрать `Edit profile`.
- В левом меню открыть `Access` -> `Password and authentication`.
- В секции `One-time password authenticator` нажать `Register authenticator`.
- Открыть приложение-аутентификатор на телефоне.
- Добавить новый аккаунт через сканирование QR-кода.
- Если камера недоступна, добавить аккаунт вручную по данным, которые показывает GitLab.
- В GitLab ввести текущий пароль.
- Ввести шестизначный код из приложения-аутентификатора.
- Нажать `Register with two-factor app`.

## Recovery codes

- После включения 2FA GitLab показывает recovery codes.
- Эти коды нужны, чтобы восстановить доступ к аккаунту, если телефон или приложение-аутентификатор недоступны.
- Каждый recovery code можно использовать только один раз.
- Сохранить recovery codes сразу после генерации.
- Не хранить recovery codes в публичных заметках, репозитории, чатах или скриншотах.
- Лучше хранить коды в менеджере паролей или другом защищенном месте.

## Проверка после настройки

- Выйти из GitLab.
- Войти заново с логином и паролем.
- Ввести код из приложения-аутентификатора.
- Убедиться, что вход работает.
- Проверить, что recovery codes сохранены.

## Если потерян доступ к приложению

- Попробовать войти с recovery code.
- Если recovery codes потеряны, обратиться к администратору GitLab или ответственному за доступы.
- Не создавать новый аккаунт без согласования, если старый аккаунт уже имеет доступ к проекту.

## Готово, если

- 2FA включена в профиле GitLab.
- Получается войти в GitLab с кодом из приложения.
- Recovery codes сохранены в безопасном месте.
